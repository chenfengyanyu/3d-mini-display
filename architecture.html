MyBetterDisplay Architecture

Claude Code 实时状态可视化 · macOS 虚拟屏管理 · 22 个动画主题

核心数据流

  • • Claude Code 触发 Hook → set-state.sh 解析 JSON
  • • curl POST /api/state 推送到 monitor.js
  • • monitor.js 内存状态机更新状态
  • • index.html 每 250ms 轮询 /api/status
  • • postMessage 驱动 iframe 主题 Canvas 动画

关键设计决策

  • • HTTP 推送取代文件轮询，零竞态
  • • AskUserQuestion → 持续黄灯，不自动转绿
  • • busy 到来立即取消 3s 倒计时
  • • 前端零计时器，严格跟随服务端
  • • Hooks 幂等注入，仅限 localhost

技术栈 & 依赖

  • • Node.js 16+（monitor.js HTTP 服务 + 状态机）
  • • Canvas 2D + requestAnimationFrame（22 主题）
  • • Bash + curl（Hook 执行器）
  • • BetterDisplay 可选（480×640 虚拟屏）
  • • macOS 12.0+ · 仅需 Node.js，无其他依赖
