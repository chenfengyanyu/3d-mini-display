Claude Code 状态机 · monitor.js 内存状态机 · 前端零计时器架构

状态触发规则

  • • UserPromptSubmit → busy（用户发消息）
  • • PreToolUse → busy（工具执行中）
  • • Stop → waiting（本 turn 结束）
  • • 3s 无新 busy 事件 → idle（真正结束）
  • • 任意 busy 到来立即取消倒计时

内存状态机设计

  • • 状态存内存，无文件 I/O 竞态问题
  • • HTTP POST 推送，毫秒级响应
  • • setTimeout 管理黄→绿倒计时
  • • clearTimeout 保证 busy 优先级最高
  • • 前端严格跟随，零自动切换逻辑

前端渲染原则

  • • 250ms 轮询 GET /api/status
  • • postMessage 驱动 16 个 iframe 主题
  • • 主题 setState() 仅做 applyState(s)
  • • ← → 切换，↑ 首个，↓ 末尾
  • • canvas.offsetWidth 适配 540×960
